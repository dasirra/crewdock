## Config Structure

The initial config is generated by `init.d/01-config.sh` (jq template) before gateway boot. At runtime, use `openclaw config get|set` to read and modify values. Changes to heartbeat, channels, and bindings are hot-reloaded without restart.

Key paths in openclaw.json:
- `agents.list[N]` - agent definitions (id, name, workspace, heartbeat)
- `agents.list[N].heartbeat.every` - heartbeat interval ("15m", "1h", "0m" to disable)
- `agents.list[N].heartbeat.target` - delivery target ("discord", "none")
- `agents.list[N].heartbeat.to` - destination ("channel:ID")
- `bindings[N]` - top-level array mapping agents to channels
- `channels.discord.accounts.<id>` - Discord account config per agent

## Config Access Examples

```bash
# Find agent index by id
openclaw config get agents.list | jq -r 'to_entries[] | select(.value.id == "forge") | .key'

# Read forge's heartbeat schedule
IDX=$(openclaw config get agents.list | jq -r 'to_entries[] | select(.value.id == "forge") | .key')
openclaw config get "agents.list[${IDX}].heartbeat.every"

# Set forge's heartbeat to every 15 minutes
openclaw config set "agents.list[${IDX}].heartbeat.every" "15m"

# Disable forge's heartbeat
openclaw config set "agents.list[${IDX}].heartbeat.every" "0m"

# Read all bindings
openclaw config get bindings
```
